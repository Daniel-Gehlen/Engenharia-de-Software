Testes - Fases do Desenvolvimento

Quando testar?

Os testes ocorrem em diferentes fases do desenvolvimento:

  • Testes de Unidade: Durante o desenvolvimento, focados em unidades individuais de código.
  • Testes de Integração: Após os testes de unidade, para garantir a integração dos módulos.
  • Testes de Validação: Garantem a satisfação dos requisitos de aceitação antes do lançamento.
  • Testes do Sistema: Testam o sistema completo, incluindo hardware, banco de dados, etc.

Tipos de Teste

  • Teste da Unidade: Testa unidades individuais de código, como funções ou métodos.
  • Teste de Integração: Testa a integração entre módulos, com abordagens top-down e bottom-up.
  • Teste de Validação: Garante a satisfação dos requisitos de aceitação e regressão.
  • Teste do Sistema: Testa o sistema completo, incluindo hardware, banco de dados e sistemas operacionais.

O que testar?

  • Funcionalidade: Garante que o software execute suas funções conforme especificado.
  • Interface: Avalia a usabilidade e a consistência da interface do usuário.
  • Desempenho: Testa o desempenho em termos de velocidade, capacidade de resposta e escalabilidade.
  • Carga (stresse): Verifica como o software lida com cargas de trabalho pesadas ou situações de estresse.
  • Usabilidade: Avalia a experiência do usuário e a facilidade de uso do software.
  • Volume: Testa o comportamento com grandes volumes de dados.
  • Segurança: Garante a resistência a ameaças e vulnerabilidades.

Como testar?

  • Teste Funcional: Avalia se o software funciona corretamente em termos de entrada e saída esperadas.
  • Teste Estrutural: Examina o código-fonte para garantir que todas as partes tenham sido testadas.
