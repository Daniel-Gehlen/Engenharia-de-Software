Conceitos de Testes de Software

Verificação e Validação

Verificação refere-se à avaliação se um sistema atende às especificações e requisitos. Validação envolve a avaliação se o software atende às necessidades do cliente.

Teste de Software e seus Processos

O teste de software é o processo de avaliação do software para identificar defeitos e garantir seu funcionamento correto. Os processos incluem planejamento, criação de casos de teste e relatórios.

Casos de Testes

Casos de teste são documentos que descrevem cenários de teste, entradas e critérios de sucesso para garantir a cobertura adequada do software.

Planejamento dos Casos de Testes

O planejamento inclui identificação de partes a serem testadas, recursos necessários e documentação dos resultados esperados.

Resultado dos Casos de Testes

Resultados registram se os testes passaram ou falharam, juntamente com problemas identificados durante a execução.

Defeito, Falha e Erro

Um defeito é uma imperfeição no software. Uma falha ocorre quando um defeito causa comportamento indesejado. Um erro é um engano humano que pode introduzir defeitos.
